const renderTemplate = (template, variables) => {
  let result = template.content
  Object.keys(variables).forEach(key => {
    const regex = new RegExp(`{${key}}`, 'g')
    result = result.replace(regex, variables[key])
  })
  return result
}

// 多渠道消息发送

  await Promise.all(promises)
}

## 🔧 重要修复记录

### 页面刷新权限问题修复 (2024-11-09)
#### 问题发现
- 页面刷新后菜单项不显示
- 权限指令v-permission未定义错误
- 用户状态未正确初始化

#### 技术解决方案
1. **权限指令实现** (`src/directives/permission.js`)
   - 实现v-permission、v-role、v-user-type权限指令
   - 支持多种权限检查方式（字符串、数组、对象）
   - 无权限时自动移除DOM元素

2. **用户状态管理增强** (`src/stores/user.js`)
   - 添加initializeAuth()方法恢复用户状态
   - 根据用户类型自动设置权限和角色
   - 支持四种用户角色的权限配置

3. **认证插件** (`src/plugins/auth.js`)
   - 应用启动时自动初始化用户认证状态
   - 避免循环依赖问题
   - 确保页面刷新后权限正常工作

4. **应用初始化优化** (`src/main.js`)
   - 注册权限指令和认证插件
   - 完善应用初始化流程

#### 修复效果
- ✅ 页面刷新后正确显示菜单项
- ✅ 权限指令正常工作
- ✅ 用户状态正确恢复
- ✅ 所有功能模块正常访问
